前言

树状数组（$ Binary Indexed Tree, BIT $，二分索引树）是一种轻量级的数据结构。它只有两种基本操作：

  1. $ add(i, 1) (1 \le i \le n) $ 对第$ i $个元素的值自增1 $ O(logn) $
  2. $ sum(i)(1 \le i \le n) ​$ 统计$ [1, i] ​$元素值的和 $ O(logn) ​$

树状数组结构

从名称中可以看到，树状数组既有树的父子节点关系，又有数组的物理空间连续存储的特性。那么如何通过数组实现树的父子节点关系呢？

我们定义，对于数组中的两个索引$ x $和$ y $，若满足$ x + 2^k = y $，其中$ k $为$ x $二进制表示中末尾0的个数，则称$ x $和$ y $之间为父子关系，其中$ y $为父节点，$ x $为子节点。

例如，对于4，其二进制表示为100，对应的k值为2，故4的父节点为8.

树状数组节点的含义

我们定义$ C_i $表示其所有节点对应元素之和。若以上图为例，那么我们可以得到：

​ C1 = A1

​ C2 = C1 + A2 = A1 + A2

​ C3 = A3

​ C4 = C2 + C3 + A4 = A1 + A2 + A3 + A4

​ C5 = A5

​ C6 = C5 + A6 = A5 + A6

​ C7 = A7

​ C8 = C4 + C6 + C7 + A8 = A1 + A2 + A3 + A4 + A5 + A6 + A7 + A8

可以发现，Ci表示了一段连续区间元素之和。易知，该区间的右端点为$ i $；从图中可以看出，左端点是沿着最左的子节点一直找到的叶节点对应下标。并且这个过程是很好实现的：

ABCD1000 → ABCD0100 → ABCD0010 → ABCD0001（模拟了“最左”）

而这个数便是$ i - 2^k + 1 $。

因此，$ C[i] = \sum_{j = i - 2^k + 1}^iA[j] $。

求和

定义 $sum(i) = \sum_{k = 1}^iA[k]$，那么如何通过Ci求sum(i)呢？

定义函数$lowbit(i) = 2^k $，其中k的含义同上，那么有：

​ $sum(i) = A[1] + A[2] + … + A[i]$

​ $= A[1] + A[2] + … + A[i - 2^k] + A[i - 2^k + 1] + … + A[i]$

​ $ = sum(i - lowbit(i)) + C[i]$

分析可知，若$ lowbit()$函数的复杂度为$ O(1) $，则该步骤的时间复杂度为$ O(logn) $。

那么如何实现$lowbit()$函数呢？

常规的实现方法是右移位运算，时间复杂度为$O(logn)$，但是我们可以把复杂度降到$O(1)$。

先复习一下补码的知识：

计算机中，数字以补码的形式存储。使用补码，可以不考虑符号位，直接实现正负数之间的运算，并且加法和减法也可以统一处理。

对于正数，其补码为原码；

对于负数，其补码为原码各位取反再加1。（均不考虑符号位）

有了补码的概念，我们就能理解$x​$&$(-x) ​$。

设$x$的二进制表示为$X0X_1…X{n-1}$，$X_0$表示符号位。则$x$和$-x$的补码分别表示为：

$x$ $(0)(X1X_2…X{n-k-1})(k个0)$ 其中$X_{n-k-1} = 1$。

$-x$ $(1)(Y1Y_2…Y{n - k - 1})(k个0)$ 其中$Y_{n-k-1} = 1$，对于其他位，有$X_i + Y_i = 1$。

因此，$x$&$(-x)$表示的就是所求的$2^k$。

更新

更新操作为$add(i, v)$，表示的就是$A[i] += v$。

但既然我们用树状数组维护原数组，那么我们需要在树状数组上进行更新。显然，当我们更新A[i]的数值时，只需要更新C[i]以及它的祖先节点；该操作的复杂度为$O(logn)$。

树状数组经典模型

改点求段

【例题1】一个长度为n(n <= 500000)的元素序列，一开始都为0，现给出三种操作：

  1. $add (x, v) $ : 给第x个元素的值加上v； ( a[x] += v )

  2. $sub (x, v)$ : 给第x个元素的值减去v； ( a[x] -= v )

  3. $sum (x, y)$： 询问第x到第y个元素的和； $ ( print \sum_{i = x}^ya[i])$

这是树状数组最基础的模型，1和2的操作就是对应的单点更新，3的操作就对应了成端求和。
具体得，1和2只要分别调用$add(x, v)$和$add(x, -v)$, 而3则是输出$sum(y) - sum(x-1)$的值。

改段求点

【例题2】一个长度为n(n <= 500000)的元素序列，一开始都为0，现给出两种操作：

  1. $add (x, y, v)​$ : 给第x个元素到第y个元素的值都加上v； ( a[i] += v, 其中 x <= i <= y )

  2. $get(x)$： 询问第x个元素的值； ( print a[x] )

这类问题对树状数组稍微进行了一个转化，但是还是可以用add和sum这两个函数来解决，对于操作1我们只需要执行两个操作，即add(x, v)和add(y+1, -v)；而操作2则是输出sum(x)的值。这样就把区间更新转化成了单点更新，单点求值转化成了区间求和。

举例说明，假设我们依次执行下列操作：

  1. $add(1, 4, 2)$
  2. $add(3, 6, 3)$
  3. $get(2)$

我们可以转化为：（其中a[]为普通数组）

  1. 初始化为0。
  2. $a[1] += 2,a[4+1] -= 2​$
  3. $a[3]+=3,a[6+1]-=3​$
  4. $sum(2)​$

通过画数轴可以很好的理解上述转化过程。

逆序数

如何定义该问题背景下树状数组的含义？

我们规定，$A[i]$表示大小为$inf-i$的元素（倒序排序）在数组中出现的个数，再用树状数组维护，易知$sum(i-1)$表示的就是大于$i$的元素的个数；我们只需依次进行更新和求和的过程并记录所有$sum$的和，最终求得的数就是总的逆序数。

可以看到，这个问题中，我们是以元素的大小决定树状数组的下标，这很容易导致空间的浪费，因此，祭上离散化大法能很好地解决这一问题。

总结

树状数组的基本操作是 单点更新和区间求和，而如何定义树状数组是相关问题的关键。

树状数组的用武之地：前缀和/差分数组、区间和、区间异或和、区间乘积
