利用ST表解决RMQ问题。

RMQ 问题

$ RMQ(Range Maximum/Minimum Query) $指，给定一个长度为$ n $的序列，多次询问区间$ [i, j] $内最值的问题。针对RMQ问题有许多处理方法，例如线段树等。本文主要讨论静态序列下的一种优良算法——ST表。

ST表

ST表是一种利用dp思想求解区间最值的算法。下面以求解最大值为例，演示ST表如何实现预处理和查询。

令$ f[i, j] $表示$ [i, i + 2^j - 1] $这段长为$ 2^j $的区间内的最大值。

初始状态：$ f[i, 0] = a_i $,即区间$ [i, i] $内的最大值为$ a_i $。

状态转移：将$ [i, i + 2^j - 1] $分成$ [i, i + 2^{j-1} - 1] $和$ [i + 2^{j-1}, i + 2^j - 1] $两段长度均为$ 2^{j-1} $的区间，那么有$ f[i, j] = max{f[i, j - 1], f[i + 2^{j-1}, j - 1]} $。

查询：若待查询区间为$ [i, j] $,那么只需找到两个区间的并集为该区间即可。

不妨取$ k = \lfloor log(j - i + 1) \rfloor $,则$ RMQ(A, i, j) = max(f[i, k], f[j - 2^k + 1, k]) ​$

这里只需证明$ i + 2^k - 1 >= len / 2 $即可。而$ j - 2^k + 1 $这个数字是怎么来的呢？设后半部分的起始位置为$ t $,则有$ t + 2^k - 1 = j $，解得$ t = j - 2^k + 1 $。

ST表预处理的时间复杂度为$ O(nlogn) $,查询的时间复杂度为$ O(1) $，空间复杂度为$ O(n^2) $。可以看到，在不支持在线修改的情况下，ST表的表现是比较不错的；但是当需要在线修改的时候，每次修改都得要更新所有的记录，因此这时我们通常采用线段树。
