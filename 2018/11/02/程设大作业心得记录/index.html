Win32 编程

windows编程基础

传统：利用API函数

  • Application Programming Interface的缩写

  • Win32 API是Windows系统与Windows应用程序间的标准程序接口

    • 为应用程序提供win系统特殊函数及数据结构

    ​ Windows应用程序利用API调用系统功能

    • 进行Windows应用程序开发必须包含windows.h头文件

交互式：利用MFC库

  • Microsoft Foundation Class Library 微软基础类库， 约有200个类
  • MFC封装了大部分API函数，并提供了一个应用程序框架，以及可视化的设计界面，简化了和标准化了Windows程序设计

目前最常用：.NET Framework/ .NET Standard

  • 微软的应用程序开发平台，支持多种语言
  • WinForm 基于控件的“属性”与“事件”开发应用
  • WPF（Windows Presentation Foundation）分离界面设计人员与开发人员的工作

窗口和句柄

窗口
  • 编写一个Windows应用程序首先应该创建一个或多个窗口
  • 应用程序的运行过程即窗口内部、窗口之间、窗口与系统之间进行数据处理与数据交换的过程
句柄
  • Windows中的句柄是一个整数（通常为32位），用来唯一标识一个Windows对象。
  • 把Windows想象成地球，在地球上寻找一个指针像经纬度，句柄就是一个身份证号，告诉Windows人的身份证号，Windows直接返回这个人给你

常用句柄类型及其说明

句柄 说明 句柄 说明
HWND 窗口句柄 HDC 设备环境句柄
HBITMAP 位图句柄 HCURSOR 光标句柄
HICON 图标句柄 HFONT 字体句柄
HMENU 菜单句柄 HPEN 画笔句柄
HFILE 文件句柄 HBRUSH 画刷句柄
HINSTANCE 当前实例句柄 HLOCAL 局部内存对象
HGLOBAL 全局内存对象

事件驱动

  • Windows程序设计是针对事件或消息的，它的执行先后顺序取决于事件的先后顺序
  • Windows环境下，应用程序启动后，系统等待用户在图形界面内的输入选择，如图标按键、键盘按键、窗口被创建、关闭、改变大小、移动等，对系统来说，这都是事件，都会产生相应的Windows消息。

消息

  • 消息是描述事件发生的信息。
  • Windows应用程序通过消息进行信息交换

Windows应用程序中的常用消息

  • 鼠标消息
    • WM_LBUTTONDOWN:产生单击鼠标左键的消息
    • WM_LBUTTONUP:放开鼠标左键时产生的消息
  • 键盘消息
    • WM_KEYDOWN:按下一个系统键时产生的消息
    • WM_KEYUP:放开一个系统键时产生的消息
  • 窗口消息
    • WM_CREATE:窗口创建时，由CreateWindows函数发出的消息
    • WM_CLOSE:关闭窗口时产生的消息
    • WM_QUIT:退出应用程序时，由PostQuitMessage函数发出的消息

Windows应用程序组成

入口函数WinMain（）

  • WinMain()函数类似C语言的main函数，是Win32应用程序入口函数
  • WinMain()函数主要完成创建主窗口，并产生和处理消息循环
  • WinMain函数的结构

窗口函数WndProc（）

  • 窗口函数WndProc（）是Windows应用程序消息处理程序
  • 窗口函数WndProc（）定义了应用程序对收到的不同消息的响应，包含了对各种可能收到的消息的处理过程
  • WndProc函数由一个或多个switch语句组成。每一条case语句对应一种消息，当应用程序接收到一个消息时，相应的case语句被激活并执行相应的相应程序模块
  • 窗口消息处理程序不予处理的所有消息给名为DefWindowProc的Windows函数默认处理
